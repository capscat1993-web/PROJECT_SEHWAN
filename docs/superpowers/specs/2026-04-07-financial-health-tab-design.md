# 재무건전성 탭 개선 — 설계 문서

**날짜:** 2026-04-07  
**범위:** `app/services/financial_health.py` + `app/templates/company.html` (재무건전성 탭)

---

## 목표

자동차부품 거래처 재무안전성 평가 엑셀 템플릿 기준(8개 지표, 100점 만점, AAA~B 6등급)으로 재무건전성 탭을 개선한다.

---

## 백엔드 변경: `financial_health.py`

### 평가 지표 (8개, 100점 만점)

| # | 영역 | 지표 | 배점 | 계산식 | 만점 기준 |
|---|------|------|------|--------|----------|
| 1 | 안전성 | 유동비율 | 20 | 유동자산 / 유동부채 × 100 | ≥150% → 20, ≥100% → 14, ≥80% → 8, 그 외 → 3 |
| 2 | 안전성 | 부채비율 | 15 | 부채총계 / 자본총계 × 100 | ≤100% → 15, ≤150% → 11, ≤200% → 6, 그 외 → 2 |
| 3 | 안전성 | 이자보상배율 | 10 | 영업이익 / 이자비용 | ≥5배 → 10, ≥2배 → 7, ≥1배 → 4, 그 외 → 0 |
| 4 | 수익성 | 영업이익률 | 20 | 영업이익 / 매출액 × 100 | ≥7% → 20, ≥3% → 14, ≥0% → 8, 그 외 → 0 |
| 5 | 수익성 | ROE | 10 | 당기순이익 / 자본총계 × 100 | ≥10% → 10, ≥5% → 7, ≥0% → 4, 그 외 → 0 |
| 6 | 성장성 | 매출액증가율 | 10 | (당기매출 - 전기매출) / 전기매출 × 100 | ≥10% → 10, ≥0% → 7, ≥-5% → 4, 그 외 → 0 |
| 7 | 활동성 | 매출채권회전일수 | 8 | 매출채권 / (매출액 / 365) | ≤45일 → 8, ≤90일 → 5, ≤120일 → 2, 그 외 → 0 |
| 8 | 현금흐름 | 영업현금흐름 | 7 | 현금흐름 섹션에서 직접 조회 | >0 → 7, >-100백만 → 4, 그 외 → 0 |

### 등급 기준

| 등급 | 점수 | 거래 권고 |
|------|------|----------|
| AAA | 85점↑ | ✅ 적극 거래 권장 |
| AA | 75~84점 | ✅ 거래 계속 / 정기 모니터링 |
| A | 65~74점 | ✅ 거래 계속 / 일부 지표 점검 |
| BBB | 55~64점 | ⚠️ 조건부 거래 / 결제조건 재검토 |
| BB | 45~54점 | ⚠️ 거래 축소 / 선결제 또는 담보 요구 |
| B | 44점↓ | 🚫 거래 재검토 / 신규 수주 중단 검토 |

### API 응답 구조 (`/api/companies/{id}/health`)

```json
{
  "period": "2023",
  "total_score": 73,
  "grade": "AA",
  "recommendation": "✅ 거래 계속 / 정기 모니터링",
  "domains": [
    {
      "name": "안전성",
      "max_score": 45,
      "score": 35,
      "items": [
        {
          "label": "유동비율",
          "value": 135.2,
          "unit": "%",
          "benchmark": "100% 이상",
          "max_score": 20,
          "score": 14,
          "grade": "B (보통)"
        }
      ]
    }
  ]
}
```

### 데이터 조회 전략

- **재무상태표**: 유동자산, 유동부채, 부채총계, 자본총계, 매출채권
- **손익계산서**: 영업이익, 매출액, 당기순이익, 이자비용
- **현금흐름분석**: 영업활동 현금흐름 (키워드 매칭)
- **성장성**: 직전 기간(period DESC LIMIT 2) 데이터를 함께 조회해 증가율 계산
- 값이 없으면 해당 지표 score=0, value=null 처리 (오류 없이 진행)

## 프론트엔드 변경: `company.html` — `loadHealth()` 함수

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│  종합 등급 카드: 등급 뱃지 + 점수 + 거래 권고   │
└─────────────────────────────────────────────────┘

┌────────────────────┐  ┌──────────────────────────┐
│  레이더 차트        │  │  영역별 점수 바 (5개)     │
│  (5개 도메인)      │  │  안전성  ████░  35/45     │
│                    │  │  수익성  ███░░  22/30     │
└────────────────────┘  └──────────────────────────┘

┌─────────────────────────────────────────────────┐
│  세부 지표 테이블                                │
│  영역 | 지표 | 값 | 벤치마크 | 점수 | 항목등급  │
└─────────────────────────────────────────────────┘
```

### 상태 처리

- 데이터 없음: "재무 데이터가 없어 분석할 수 없습니다" 메시지
- 일부 지표 누락: 해당 행 값 칸에 "-" 표시, score=0으로 합산
